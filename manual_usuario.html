Capítulo 1

Introducción al Sistema

Conceptos generales, acceso, navegación y roles de usuario

CORFIEM ERP es un sistema de gestión empresarial diseñado para administrar el ciclo completo del negocio: desde la captación de prospectos y la generación de cotizaciones, hasta la ejecución de proyectos, el control de incidencias, la capacitación del equipo y la generación de reportes gerenciales.

El sistema está organizado en módulos accesibles desde la barra lateral izquierda. Cada módulo corresponde a un área funcional de la empresa.

1.1 Acceso e inicio de sesión

Ingresar al sistema desde cualquier navegador moderno (Chrome, Edge, Firefox):

1
Abrir la URL del sistema Ingresar http://localhost/Corfiem_Cesar en el navegador (o la URL provista por el administrador para el entorno de producción).
2
Ingresar credenciales Escribir el correo electrónico y la contraseña. Las credenciales de acceso son asignadas por el administrador del sistema.
3
Hacer clic en "Iniciar Sesión" El sistema redirige automáticamente al Dashboard si las credenciales son correctas.
Contraseña segura La contraseña debe tener al menos 8 caracteres, combinando letras, números y símbolos. Ante cualquier problema de acceso, contactar al administrador del sistema.

1.2 Roles y permisos

El sistema maneja cuatro roles de usuario. Cada rol define qué acciones puede realizar el usuario dentro de cada módulo.

Rol Acceso general Restricciones
Admin Acceso total al sistema: crear, editar y eliminar en todos los módulos. Ninguna. Puede gestionar usuarios y ver auditoría completa.
Gerente Crear y editar proyectos, CRM completo, ver BI y reportes. No puede eliminar registros ni gestionar usuarios.
Usuario Consultar información, agregar notas e interacciones. No puede eliminar ni crear cursos/usuarios. Edición limitada.
Consultor Lectura de proyectos asignados y CRM. Solo lectura en la mayoría de módulos.
Nota sobre permisos A lo largo de este manual, las secciones restringidas a un rol específico están marcadas con la etiqueta del rol correspondiente. Las funcionalidades sin marca están disponibles para todos los roles.

1.3 Navegación general

La interfaz está dividida en tres zonas principales:

Barra lateral (izquierda)

Acceso a todos los módulos del sistema. El ítem activo aparece resaltado en azul.

  • Dashboard
  • Actividades Previas
  • CRM — Clientes
  • Proyectos
  • Puesta en Marcha
  • Reportes BI
  • Capacitación
  • Auditoría
Barra superior (topbar)

Muestra el título de la sección actual, el nombre del usuario y el botón de cierre de sesión.

Área de contenido

Ocupa el área central. Contiene tablas, formularios, gráficos y modales según el módulo activo.

Las operaciones de creación y edición se realizan en ventanas modales que aparecen sobre la pantalla sin salir de la página actual. Los mensajes de confirmación aparecen como notificaciones toast en la esquina inferior derecha.

Dashboard — Panel Principal

Vista general del estado del negocio en tiempo real

El Dashboard es la pantalla de inicio del sistema. Ofrece una vista consolidada de los indicadores más importantes sin necesidad de ingresar a cada módulo.

2.1 Indicadores KPI

La fila superior muestra seis tarjetas de indicadores clave (KPI) calculados en tiempo real:

IndicadorDescripción
Proyectos ActivosCantidad de proyectos en estado diferente a Completado, Cancelado o En Pausa.
Proyectos CompletadosProyectos con estado "Completado".
Clientes ActivosClientes con al menos un proyecto activo.
Incidencias AbiertasIncidencias con estado "Abierta" (no resueltas). Se ordena por severidad.
Presupuesto ActivoSuma del presupuesto de todos los proyectos activos (en S/).
Avance PromedioPorcentaje medio de avance de todos los proyectos activos.

2.2 Proyectos recientes e incidencias

Proyectos recientes

Muestra los últimos 6 proyectos creados o actualizados. Por cada proyecto se visualiza:

  • Nombre y código del proyecto.
  • Nombre del cliente.
  • Estado actual (badge de color).
  • Barra de progreso con el porcentaje de avance.
  • Fecha estimada de finalización.

Incidencias críticas

Listado de las 5 incidencias abiertas más graves, ordenadas de mayor a menor severidad (Crítica → Alta → Media → Baja). Permite identificar rápidamente los problemas que requieren atención.

Actividad reciente del sistema

Registro de las últimas 8 acciones realizadas en el sistema por cualquier usuario: creaciones, modificaciones, eliminaciones e inicios de sesión. Útil para supervisión del equipo.

Actualización de datos Los datos del dashboard se actualizan al recargar la página. No existe actualización automática en segundo plano.

3.4 Cotizaciones

Desde el módulo de Previas se pueden crear cotizaciones asociadas a un prospecto específico. Hacer clic en "+ Nueva Cotización" para:

1
Seleccionar el prospecto Buscar el prospecto por empresa, nombre de contacto o RUC usando el buscador del modal.
2
Confirmar los datos El sistema muestra la razón social, RUC y estado del prospecto seleccionado.
3
Crear la cotización Al confirmar, se crea el registro de cotización y el sistema redirige al editor de cotización donde se agregan los ítems, montos y condiciones.
Estados de cotización Las cotizaciones pueden estar en: Borrador, Enviada, Aceptada, Rechazada o Vencida. El sistema aplica automáticamente el IGV (18%) si la cotización tiene activada la opción Aplica IGV.

4.4 Eliminar cliente Solo Admin

La eliminación de un cliente es una operación destructiva e irreversible. Solo los usuarios con rol Admin pueden realizarla. El proceso exige confirmación en tres pasos:

1
Clic en el ícono de eliminar Aparece un modal de advertencia indicando el nombre del cliente y los proyectos asociados que se eliminarán.
2
Revisar el impacto El modal muestra cuántos proyectos, entregables y archivos serán eliminados permanentemente.
3
Escribir el nombre del cliente para confirmar Se debe escribir exactamente la razón social del cliente en el campo de confirmación antes de poder proceder.
Eliminación en cascada Al eliminar un cliente se eliminan también: todos sus proyectos, entregables, archivos adjuntos físicos, pagos y documentos de conformidad. Esta acción no se puede deshacer.
Capítulo 5

Proyectos

Planificación, seguimiento de avance, entregables y documentos de proyecto

El módulo de Proyectos permite gestionar el ciclo de vida completo de cada proyecto: desde la planificación inicial hasta el cierre, controlando entregables, avance, presupuesto y archivos.

5.1 Cuadrícula de proyectos

Los indicadores de la parte superior muestran:

  • Total proyectos registrados en el sistema.
  • Proyectos activos: en estados distintos a Completado, Cancelado o En Pausa.
  • Completados: proyectos finalizados exitosamente.
  • Presupuesto activo: suma presupuestada de proyectos activos (S/).
  • Avance promedio: porcentaje medio de progreso de proyectos activos.

Cada proyecto se muestra como una tarjeta con:

  • Código y nombre del proyecto.
  • Nombre del cliente.
  • Badges de estado y prioridad.
  • Barra de progreso con porcentaje de avance.
  • Contador de entregables completados (ej. 4/7).
  • Presupuesto y fecha estimada de finalización.

Filtros y ordenamiento

  • Búsqueda: código, nombre o cliente.
  • Cliente: filtrar por cliente específico.
  • Estado y Prioridad: selección múltiple.
  • Ordenar por: código, nombre, avance, presupuesto, fechas, entre otros.
  • Paginación: 15 proyectos por página.

5.2 Detalle del proyecto

Al hacer clic en una tarjeta de proyecto se accede a la vista de detalle. Esta página tiene varias pestañas para organizar la información:

PestañaContenido
InformaciónDatos generales del proyecto: descripción, alcance, responsable, fechas, presupuesto, costo real y avance.
EntregablesLista de entregables con estado, porcentaje y archivos adjuntos.
ActividadesTareas o actividades asociadas al proyecto.
DocumentosDocumentos administrativos del proyecto (contratos, actas, etc.).

Edición de información del proyecto

En la pestaña Información se pueden editar:

Estado
Cambiar la etapa del proyecto (ver estados en sección 5.4).
Responsable
Usuario del equipo a cargo del proyecto.
Fecha Inicio
Fecha oficial de inicio del proyecto.
Fecha Fin Estimada
Fecha planificada de cierre.
Presupuesto
Monto presupuestado en soles (S/).
Costo Real
Costo efectivo incurrido hasta la fecha.
Avance %
Porcentaje de avance manual (se recalcula automáticamente cuando se actualizan entregables).

5.3 Entregables y archivos

Los entregables son los productos o resultados concretos que debe producir el proyecto. El avance global del proyecto se calcula automáticamente en base al porcentaje de cada entregable.

Campos de cada entregable

Nombre
Descripción del entregable (ej. "Informe técnico final").
Estado
Pendiente En proceso Completado
Porcentaje de avance
De 0 a 100%. Afecta el avance global del proyecto.
Archivos adjuntos
Documentos, imágenes, PDFs o ZIPs relacionados al entregable.
Fecha de ampliación
Si el entregable tuvo un plazo extendido, aparece una alerta amarilla con la fecha.

Carga de archivos

Formatos permitidos: PDF, JPG, PNG, DOC, DOCX, ZIP.

Tamaño máximo: 20 MB por archivo (archivos ZIP: hasta 200 MB).

Los archivos se almacenan en el servidor y pueden descargarse en cualquier momento desde la lista de entregables.

Cálculo automático de avance El porcentaje de avance del proyecto se recalcula automáticamente cada vez que se modifica el porcentaje de un entregable. No es necesario actualizar el avance del proyecto manualmente.

5.4 Estados y ciclo de vida del proyecto

EstadoDescripción
PlanificaciónProyecto en fase inicial; aún no se han iniciado las actividades.
En CursoProyecto en ejecución activa.
ValidaciónEntregables completados; pendiente de revisión y aceptación por el cliente.
CompletadoProyecto finalizado y aceptado.
CanceladoProyecto interrumpido definitivamente.
En PausaProyecto temporalmente suspendido.

8.4 Cuestionarios y evaluaciones

La pestaña Cuestionarios permite crear y gestionar evaluaciones para el curso.

Crear cuestionario Solo Admin

Clic en "Nuevo Cuestionario". Se abre el editor donde se pueden agregar preguntas de:

  • Opción múltiple: una o varias respuestas correctas.
  • Verdadero / Falso.
  • Respuesta abierta (ensayo): texto libre, calificada manualmente.

Cada pregunta tiene un valor en puntos. Se define la calificación mínima para aprobar el cuestionario.

Tomar evaluación (participante)

Al acceder a la evaluación, el participante responde las preguntas y envía sus respuestas. El sistema muestra el resultado inmediatamente para preguntas objetivas, o tras revisión manual para preguntas de ensayo. La calificación queda registrada en la ficha de inscripción.

Capítulo 10

Preguntas Frecuentes

Respuestas a dudas comunes sobre el uso del sistema

¿Por qué no veo el botón de eliminar en algunos registros?

Los botones de eliminación están disponibles únicamente para usuarios con rol Admin. Si no ves este botón, es porque tu rol actual no tiene ese permiso. Contacta al administrador del sistema si necesitas eliminar un registro.

¿Cómo actualizo el avance de un proyecto?

El avance del proyecto se recalcula automáticamente al actualizar el porcentaje de sus entregables. Accede al detalle del proyecto → pestaña Entregables → edita el porcentaje de cada entregable. El avance global se actualizará automáticamente.

¿Puedo recuperar un registro eliminado?

No directamente desde la interfaz. Las eliminaciones son permanentes. Sin embargo, el registro de auditoría (módulo Seguridad) guarda un snapshot JSON del registro antes de eliminarlo. El administrador puede revisar esos datos para recuperar la información manualmente.

¿Cómo convierto un prospecto en cliente?

Hay dos formas:

  1. Desde el módulo CRM → Nuevo Cliente: al escribir la razón social, el sistema sugiere prospectos existentes y auto-completa los datos.
  2. Desde el detalle del prospecto: si el estado es Aceptado, aparece la opción de conversión directa.

Al convertir, el prospecto queda vinculado al cliente y se crea automáticamente un proyecto inicial.

¿Cuál es el tamaño máximo de archivo que puedo subir?

El límite general es 20 MB por archivo para formatos PDF, JPG, PNG, DOC y DOCX. Los archivos ZIP tienen un límite especial de 200 MB.

¿El sistema aplica el IGV automáticamente en las cotizaciones?

Sí. Si una cotización tiene activada la opción Aplica IGV, el sistema calcula automáticamente el 18% de IGV y lo suma al subtotal para obtener el total. Este cálculo se actualiza cada vez que se modifica la cotización o sus ítems.

¿Cómo cierro la sesión?

Hacer clic en el nombre de usuario en la barra superior derecha y seleccionar "Cerrar sesión". El sistema registra el evento de cierre en el log de auditoría.

¿Qué hago si olvidé mi contraseña?

Contactar al administrador del sistema para que restablezca la contraseña desde el módulo de Seguridad → Gestión de Usuarios → Editar usuario.

¿Los datos del Dashboard se actualizan en tiempo real?

Los datos se actualizan al cargar o recargar la página (F5). No hay actualización automática en segundo plano. Para ver datos actualizados, recargar la página del Dashboard.
